# Product

## Register

product

## Users

Godploy is used by solo operators and small teams running deployable services on their own VPS infrastructure. They are usually inside operational workflows: creating projects, managing project instances, wiring services, checking runtime state, and recovering from deploy or database issues without leaving the dashboard.

## Product Purpose

Godploy is a lightweight, self-hosted PaaS for managing projects, isolated project instances, application services, and predefined database services. Success means users can understand runtime boundaries quickly, perform common deploy and service-management actions safely, and trust the UI during demos and production-like operations.

## Brand Personality

Simple, easy-to-use, scales-with-you. The interface should feel clear and practical for first-time users while still holding up for deeper infrastructure workflows.

## Anti-references

Do not look like a typical purple SaaS dashboard. Avoid brutalism, decorative texture, soft overly-rounded surfaces, and heavy card-based layouts when plain form structure would be clearer.

## Design Principles

- Make runtime boundaries obvious before asking users to act.
- Favor plain, direct controls over decorative containers.
- Keep infrastructure workflows calm, readable, and hard to misuse.
- Show enough technical detail for operators without making the UI feel dense.
- Design for growth: simple first use, consistent structure for more services later.

## Accessibility & Inclusion

Target WCAG AA basics: readable contrast, keyboard-usable controls, clear focus states, accessible labels, and reduced-motion-safe interaction patterns.
