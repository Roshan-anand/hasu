# Godploy — Project Knowledge

Lightweight, single-binary, self-hosted PaaS — alternative to Dokploy/Coolify/Netlify/Vercel.

**Stack:** Go (Echo v5) · SvelteKit 5 SPA (embedded in binary) · SQLite via sqlc · BadgerDB (log buffer) · Docker · Traefik

See `docs/prd.md`, `docs/mvp_roadmap.md`, and `docs/design_docs/` for deeper context.

## Quickstart

Prereqs: `docker` v29.x, `docker-compose`, `openssl`, `bun`, `go` 1.25+, `sqlc`.

```bash
# one-time local TLS certs
mkdir -p ./dynamic/certs
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout ./dynamic/certs/local.key -out ./dynamic/certs/local.crt \
  -subj "/CN=*.godploy.localhost"

# add to /etc/hosts
# 127.0.0.1  *.godploy.localhost

cp .env.example .env
make cloud-tunnel      # copy https://<...>.trycloudflare.com into SERVER_PUBLIC_URL in .env
make setup             # install deps + build web + setup traefik service
make dev-start         # start dev containers (web + server) via docker compose watch
```

URLs:

- Web: `https://localhost:3000`
- Server: `https://localhost:8080` (or the cloudflared URL)
- Traefik dashboard: `https://traefik.godploy.localhost` (user: `godploy`, pass: `godploy`)

## Common Commands (from repo root)

| Task                         | Command                                                    |
| ---------------------------- | ---------------------------------------------------------- |
| Install all deps             | `make install`                                             |
| Install web only             | `make install-web` (uses `bun`)                            |
| Install server only          | `make install-server` (`go mod tidy`)                      |
| Start dev services           | `make dev-start`                                           |
| Stop dev services            | `make dev-stop`                                            |
| Remove traefik swarm service | `make services-rm`                                         |
| Tail logs                    | `make web-logs` / `make server-logs` / `make traefik-logs` |
| Format all                   | `make format` (`format-server` / `format-web`)             |
| Frontend typecheck           | `make check` (runs `bun check` in frontend)                |
| Run Go tests                 | `make test`                                                |
| Regenerate sqlc code         | `make generate`                                            |
| Build production binary      | `make build` (builds web + `bin/godploy`)                  |
| Run built binary             | `make start`                                               |
| Reset local data             | `make reset`                                               |
| Clean                        | `make clean` / `clean-web` / `clean-server`                |
| Fix .svelte-kit perms        | `make permission`                                          |

Frontend-only (in `frontend/`): `bun run dev`, `bun run build`, `bun run check`, `bun run lint`, `bun run format`, `bun run test:unit`.

## Architecture

### `backend/` — Go (Echo v5), clean-architecture-ish

- `cmd/server/main.go` — HTTP server entrypoint
- `cmd/setup/main.go` — local-dev CLI (`setup`, `dev-start`, `dev-stop`, `*-logs`)
- `internal/config/` — loaders for core, http, docker, sqlite, badger
- `internal/handlers/` — HTTP handlers (`auth`, `org`, `service`, `app_service`, `psql_service`, `github`, `health`)
- `internal/middleware/` — auth, CORS, rate-limit, etc.
- `internal/routes/` — route registration
- `internal/lib/` — session, password, csrf, uuid/id gen, ghinstall, sse, shared `types`
- `internal/jobs/` — background workers
  - `deployment/` — `pull`, `build`, `deploy` stages with internal `queue`
  - `logbroker/` — streams container/deployment logs (SSE), BadgerDB-backed buffer
- `internal/db/` — **generated by sqlc, do not edit by hand**
- `sqlite/migrations/` — `golang-migrate` up/down SQL
- `sqlite/query/*.sql` — sqlc query sources (`user`, `session`, `organisation`, `service`, `github`)
- `frontend/embed.go` — embeds built SvelteKit SPA into the Go binary
- `integration_tests/` — Go integration tests (`auth`, `org`, `psql_service`)

Data: SQLite for metadata, BadgerDB for log buffering, Docker socket for container ops, Traefik for ingress (dynamic config in `/dynamic`).

### `frontend/` — SvelteKit 5 SPA (adapter-static)

- `src/routes/(auth)/` — login, register
- `src/routes/(protected)/(core)/` — authed app ( service, git, members, layout)
- `src/lib/components/ui/` — shadcn-svelte primitives (bits-ui under the hood)
- `src/lib/components/services/` — service detail views (`app`, `psql`) + list
- `src/lib/features/<domain>/` — per-domain `query.ts`, `mutation.ts`, `store.svelte.ts`, `type.ts` (`auth`, `projects`, `services`, `deployments`, `git`, `global`)
- `src/lib/axios.ts`, `query.ts`, `utils.ts` — shared http/query client wiring
- Uses TanStack Query (`@tanstack/svelte-query`), TanStack Form, Zod, Tailwind v4, `@lucide/svelte`, `mode-watcher`, `svelte-sonner`.

## Conventions

- **Package manager (frontend):** `bun` only. Install shadcn pieces via `bun x shadcn-svelte@latest add <component>`.
- **SQL/DB:** Never edit `backend/internal/db/*`. Add migrations in `backend/sqlite/migrations/` and queries in `backend/sqlite/query/*.sql`, then `make generate`.
- **Patterns:** Mirror existing neighbors — handlers, queries, UI components, stores, and query/mutation hooks should follow adjacent files’ style.
- **Comments:** Skip for routine handlers/queries. Add short "what & why" comments for new patterns, utilities, or AI-generated non-obvious code.
- **Knowledge capture:** When introducing a new concept/pattern, append a note to `docs/queries.md`.
- **Interaction mode (see `AGENTS.md`):** Owner is in a learning phase — default to Socratic guidance, no spoon-feeding. When prompt starts with `*tds`, switch to direct-execution mode and just implement.
- **Routing:** SvelteKit group routes `(auth)` and `(core)` separate unauth vs authed layouts.
- **Server-state:** TanStack Query is the source of truth for server data; prefer it over ad-hoc stores for remote data.
- **Styling:** Tailwind v4 via `@tailwindcss/vite`; use shadcn primitives over hand-rolled components.

## Gotchas

- `.svelte-kit/` can end up root-owned after docker compose runs — fix with `make permission`.
- `SERVER_PUBLIC_URL` in `.env` must be set (cloudflared tunnel URL) before `make setup` for GitHub App callbacks to work locally.
- `make reset` wipes `backend/data/*` and `./data/*` — includes SQLite DB and Badger logs.
- Frontend is embedded into the Go binary at build time (`backend/frontend/embed.go`), so production changes to the SPA require `make build-web` before `make build-bin`.
- Traefik is run as a Docker **swarm service** (`godploy_traefik`); `make services-rm` removes it.
- SQLite foreign keys must be enabled at connection time (see `internal/config/sqlite.go`); recent bugfix enforced this.
- sqlc generation overwrites `internal/db/` — any manual edits there will be lost.
